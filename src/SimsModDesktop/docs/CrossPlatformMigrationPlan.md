# 跨平台迁移计划（重构后版本）

## 1. 当前状态

跨平台迁移已从“规划阶段”进入“收敛阶段”：

* 桌面应用主路径为 .NET 8 + Avalonia。
* 文件操作与哈希能力通过 `IFileOperationService`、`IHashComputationService` 抽象。
* 统一文件转换引擎已承接 `flatten/normalize/merge/organize`。
* Tray 依赖、Package 解析、纹理处理均有独立 .NET 组件。

PowerShell 脚本链路已从仓库主链路移除，桌面端完全由 .NET 运行时承载。

---

## 2. 目标

1. 保持现有跨平台能力稳定（Windows 优先，macOS 持续验证）。
2. 稳定桌面端各功能路径的一致性与可测试性。
3. 将历史兼容缝隙（如 `NoOp*UseCase`）收敛为可维护的正式用例实现。

---

## 3. 收敛工作项

### 3.1 执行与用例层
* 对高频路径落地真实用例服务：执行、预览、索引、导出。
* 统一结果模型，减少字符串拼接状态传递。

### 3.2 文件与路径行为一致性
* 继续补齐 Windows/macOS 在路径规范、删除语义、大小写行为上的测试覆盖。
* 对大目录批处理路径做压力验证和回归基线。

### 3.3 工程与发布
* 持续收敛文档，保证描述仅覆盖当前桌面主链路。
* 在 CI 中保持跨平台构建和核心测试矩阵。

---

## 4. 风险与缓解

1. 行为漂移风险：历史迁移阶段遗留约定与当前实现不一致。  
缓解：建立固定回归样本并持续更新文档与测试基线。

2. 迁移缝隙长期存在：`NoOp*UseCase` 未收敛会造成理解成本。  
缓解：按调用频度分批替换，先替换核心用户路径。

3. 性能回归风险：跨平台 I/O 与并发参数差异导致吞吐波动。  
缓解：为关键动作维护平台分组基准测试。

---

## 5. 完成判定

* 主要用户路径在 Windows 与 macOS 均可稳定运行。
* 核心动作具备可复现的端到端测试基线。
* 文档与代码结构一致，不再引用旧路由/旧策略组件。
